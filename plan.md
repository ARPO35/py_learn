# Python 学习 CLI 开发计划

本文档描述 `py_learn` / `pl` 的第一版开发计划。

## 第一版目标

实现一个可安装的 Python CLI 程序，安装后通过以下命令使用：

```bash
pl
```

第一版覆盖核心学习闭环：

1. 扫描原始练习。
2. 列出章节和练习。
3. 在当前目录创建或恢复练习。
4. 保存答题进度。
5. 运行调试用例。
6. 运行提交验证用例。
7. 提交通过后自动保存。

## 明确不做

第一版不做：

- 不使用 `pl go`，使用 `pl resume`。
- 不使用 `pl commit`，使用 `pl submit`。
- 不支持传统 stdin/stdout 整文件判题。
- 不做强沙箱隔离。
- 不限制学生代码的文件系统能力。
- 不把验证文件复制到答题目录。
- 不通过通配符清理答题目录。
- 不让扫描器写回 `exercise.toml`。
- 不要求练习放在固定目录深度。

## 技术选择

- 包管理与构建：`pyproject.toml`。
- CLI 入口：标准库 `argparse` 优先，避免第一版引入额外依赖。
- TOML 读取：Python 3.11+ 使用标准库 `tomllib`。
- 用例执行：直接导入答题目录中的 `a.py` 并调用 `py_learn`。
- 耗时统计：`time.perf_counter()`。
- 内存统计：`tracemalloc` 近似统计峰值内存。
- 内容来源：内置内容目录 + 用户配置内容目录。

## 命令设计

### `pl ls [章节]`

无参数时列出章节树：

```text
A. 循环
B. 基础面向对象
|-- B1. 了解类(class)
|-- B2. 调用方法
```

指定章节时列出该章节下的练习：

```text
B2. 调用方法
|-- 1. self 函数
```

### `pl new <练习编号>`

从原始练习重新开始。

规则：

- 当前答题目录承载另一个练习时，先保存当前练习。
- 当前答题目录承载同一个练习时，不保存当前未保存修改。
- 不删除旧答题进度快照。
- 从原始练习复制学习材料和答题模板。

### `pl resume <练习编号>`

恢复练习的上一次答题进度快照。

规则：

- 当前答题目录承载另一个练习时，先保存当前练习。
- 当前答题目录承载同一个练习时，不保存当前未保存修改。
- 直接用快照覆盖当前 CLI 管理文件。
- 如果目标练习没有快照，提示用户使用 `pl new <练习编号>` 开始。

### `pl save`

保存当前答题目录内容，覆盖当前练习的答题进度快照。

规则：

- 使用 `.pl/.plignore` 排除学习材料和内部状态。
- 默认不保存 `q.md`。
- 保存用户创建的辅助文件。

### `pl next`

跳到下一个练习。

规则：

- 先保存当前练习。
- 根据章节顺序和练习顺序查找下一个练习。
- 如果下一个练习有快照，则恢复快照。
- 如果没有快照，则从原始练习开始。

### `pl run`

运行调试用验证用例。

规则：

- 不自动保存。
- 使用原始练习中的验证文件。
- 选择 `RUN_CASES`。
- 调用答题目录中的 `py_learn`。
- 输出每个用例的输入、期望、实际、异常和耗时。

### `pl submit`

运行提交验证用例。

规则：

- 使用原始练习中的验证文件。
- 选择 `SUBMIT_CASES`。
- 输出测试结果、总耗时和近似峰值内存。
- 所有用例通过后自动执行 `pl save`。
- 失败时不自动保存。

## 开发阶段

### 阶段 1：项目骨架

交付：

- `pyproject.toml`
- `src/py_learn/cli.py`
- `pl` 命令入口
- 基础帮助信息

验收：

```bash
pl --help
```

能输出命令列表。

### 阶段 2：练习扫描

交付：

- 递归扫描内容目录。
- 识别 `type = "chapter"` 和 `type = "exercise"`。
- 合并继承元数据。
- 检测重复 Exercise ID。
- 构建章节树和练习列表。

验收：

- 支持扁平练习目录。
- 支持嵌套练习目录。
- 下层字段覆盖上层字段。
- 冲突 ID 报错。

### 阶段 3：`pl ls`

交付：

- 无参数列出章节。
- 指定章节列出练习。
- 按 `order` 排序。

验收：

```bash
pl ls
pl ls B1
```

输出结构符合设计。

### 阶段 4：答题目录管理

交付：

- `.pl/state.json`
- `.pl/manifest.json`
- `.pl/.plignore`
- 安全复制学习材料和答题模板。
- 安全清理上一练习的 CLI 管理文件。

验收：

- 切换练习不会删除未记录的用户文件。
- 验证文件不会复制到答题目录。
- `.plignore` 会复制进 `.pl/`。

### 阶段 5：答题进度快照

交付：

- 全局快照存储。
- `pl save`。
- `pl new`。
- `pl resume`。

验收：

- `new` 当前练习不保存未保存修改。
- `resume` 当前练习不保存未保存修改。
- 跨练习 `new` / `resume` 会先保存当前练习。
- `new` 不删除旧快照。
- `resume` 可以恢复旧快照。

### 阶段 6：验证执行

交付：

- 加载答题目录 `a.py`。
- 获取 `py_learn`。
- 加载原始练习 `cases.py`。
- 捕获返回值、stdout、异常。
- 支持 `args` / `kwargs`。
- 支持 `expected_return` / `expected_stdout`。

验收：

- `pl run` 使用 `RUN_CASES`。
- `pl submit` 使用 `SUBMIT_CASES`。
- 返回值判题正常。
- 输出判题正常。
- 异常报告清晰。

### 阶段 7：性能统计和提交保存

交付：

- 单用例耗时。
- 总耗时。
- `tracemalloc` 近似峰值内存。
- `pl submit` 通过后自动保存。

验收：

- 提交失败不保存。
- 提交通过保存快照。
- 报告展示耗时和内存。

### 阶段 8：`pl next`

交付：

- 根据章节和练习顺序计算下一个练习。
- 自动保存当前练习。
- 有快照则恢复，没有快照则新建。

验收：

- 同章节下可跳下一题。
- 跨章节可跳下一题。
- 最后一题时给出明确提示。

### 阶段 9：内容来源配置

交付：

- 内置内容目录扫描。
- 用户额外内容目录配置。
- 多来源合并。
- 重复 ID 冲突提示。

验收：

- 无用户配置时可使用内置练习。
- 配置额外目录后可发现用户练习。
- 重复 ID 时阻止继续并显示冲突路径。

## 测试计划

- 重复 Exercise ID 检测。

- `pl run` 执行调试用例。

- 验证文件不出现在答题目录。
- `new` 不删除旧快照。
- 当前练习执行 `resume` 不保存未保存修改。
- 当前练习执行 `new` 不保存未保存修改。

## 风险与处理

### 学生代码死循环

第一版不做强沙箱和超时隔离。若执行卡住，用户手动中断。

后续可引入子进程超时执行。

### 内存统计不精确

第一版使用 `tracemalloc` 做 Python 分配层面的近似峰值内存统计。

报告中应描述为“近似峰值内存”。

### 内容来源冲突

多个内容目录可能声明相同 Exercise ID。

第一版直接报错，不做覆盖。

### 练习元数据不完整

扫描时发现缺少必填字段，应报错并显示对应 `exercise.toml` 路径。

## 第一批内置练习建议

建议用少量练习覆盖功能：

```text
A. 基础语法
|-- A.1 返回两个数之和
|-- A.2 打印问候语

B. 基础面向对象
|-- B1. 了解类(class)
    |-- B1.1 定义简单类
    |-- B1.2 self 函数
```

其中：

- `A.1` 验证 `expected_return`。
- `A.2` 验证 `expected_stdout`。
- `B1.2` 验证继承章节元数据和嵌套目录扫描。
