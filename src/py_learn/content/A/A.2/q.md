# 练习 A.2：认识变量

想象你的书桌上有一堆贴了标签的盒子：你可以在盒子里放东西，然后通过标签找到它。**变量**就是这样一个贴标签的盒子——**变量名**是标签，**变量值**是盒子里的东西。

## 概念讲解

### 赋值：给盒子贴标签

**赋值**的语法就是把等号右边的值放进左边名字代表的盒子里：

```python
x = 10  # 把整数 10 放进名叫 x 的盒子里
```

之后只要写 `x`，Python 就知道你在说 `10`。

同一个盒子可以换东西——**变量可以重新赋值**，旧的值会被覆盖：

```python
message = "早上好"   # 盒子 message 里放了 "早上好"
message = "下午好"   # 现在 message 里换成了 "下午好"
```

上面两行分开看：第一行创建变量并赋值，第二行**重新赋值**。重新赋值后，`message` 就不再是 `"早上好"` 了。

### f-string：把变量嵌入字符串

**f-string**（格式化字符串）可以把变量的值直接塞进一段文字中。

语法结构（模板，不是完整程序）：

```python
f"任意文字{变量名}任意文字"
```

拆开来看：

- 开头的 `f` —— **必须写**，告诉 Python 「后面的字符串里有变量要替换」
- 花括号 `{ }` —— 里面放变量名，Python 会用变量的值替换它
- 花括号外 —— 普通文字，原样保留

举个概念演示：

```python
f"你好，{name}！"  # 假设 name 的值是 "小红"
```

`{name}` 会被替换成 `"小红"`，最终结果是：

```
你好，小红！
```

## 常见错误

### 1. 变量名拼写不一致

```python
name = "小明"
print(nme)  # 错误！nme 没有定义过，Python 抛出 NameError
```

赋值时叫 `name`，使用时必须写**完全一样**的拼写。Python 区分大小写：`Name`、`name`、`NAME` 是三个不同的变量名。

### 2. 忘记 f 前缀

```python
"你好，{name}"   # 错误！少了 f，花括号被当作普通文字输出
f"你好，{name}"  # 正确 —— 花括号内的 name 会被替换成变量的值
```

不加 `f` 的字符串就是普通字符串，花括号不会被特殊处理。

## 题目要求

编写函数 `py_learn(name)`，接收一个字符串参数 `name`，返回一句问候语。

- 函数签名：`def py_learn(name):`
- 返回值是一个**加了 f 前缀的字符串**，其中用花括号嵌入参数 `name`
- 返回类型：`str`

## 提示

- `return` 后面直接写 f-string 即可，不需要额外的变量
- f-string 的写法：`f` 开头，花括号 `{}` 里写变量名，外面是普通文字
- 参数 `name` 在函数体内可以直接使用，它就是调用者传进来的字符串
