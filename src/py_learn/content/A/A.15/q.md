# 找不可拆分的「积木」

学习概念就像玩乐高——有些积木块是最小的单元，不能再拆成更小的了。质数也是这个道理。

## 概念讲解

**质数**（也叫素数）是大于 1 的自然数中，只能被 **1 和它本身**整除的数。换句话说，除了 1 和它自己，没有别的整数能整除它。

- 质数：2, 3, 5, 7, 11, 13, 17, 19, 23……
- 合数：4（能被 2 整除）, 6（能被 2、3 整除）, 8, 9……

判断一个数 `n` 是不是质数，只需要从 2 开始，一直检查到 `n-1`，看看有没有能整除 `n` 的。如果都没有，它就是质数。

## 代码示例

下面是一个初学者友好的实现：

```python
def is_prime(n):
    if n < 2:
        return False
    for i in range(2, n):
        if n % i == 0:
            return False
    return True
```

- 检查 `is_prime(7)`：2 不整除、3 不整除、4 不整除、5 不整除、6 不整除 → 返回 `True`
- 检查 `is_prime(9)`：2 不整除、3 整除（9 % 3 == 0）→ 返回 `False`

优化建议：其实不需要检查到 `n-1`，只需到 `√n`（平方根）即可。因为如果 `n = a × b`，那么 `a` 和 `b` 中至少有一个小于等于 `√n`。

```python
import math

def is_prime_optimized(n):
    if n < 2:
        return False
    for i in range(2, int(math.sqrt(n)) + 1):
        if n % i == 0:
            return False
    return True
```

两种思路都可以，练习中我们用第一种（检查到 n-1），更容易理解。

## 常见错误

1. **把 1 当作质数**：1 不是质数！质数的定义要求大于 1。
2. **忘记处理 2**：2 是最小的质数，检查范围 `range(2, 2)` 是空的，循环不会执行，直接走到 `return True`，结果正确——但你得明白为什么。
3. **循环条件写错为 `range(1, n)`**：这会用 1 去整除，`n % 1 == 0` 永远成立，导致所有数都被判定为非质数。

## 题目要求

补全函数 `py_learn(n)`：

- **输入**：一个整数 `n`
- **输出**：如果 `n` 是质数返回 `True`，否则返回 `False`
- **约束**：不需要考虑 `n` 小于等于 1 的情况，但最好兼容处理
- **目标**：综合运用函数定义、for 循环、条件判断和 return 语句

## 提示

1. 先用 `if n < 2: return False` 过滤掉 1 和负数
2. 用 `for i in range(2, n):` 遍历检查
3. 如果 `n % i == 0`，说明 `i` 能整除 `n`，返回 `False`
4. 循环结束后都没有找到除数，返回 `True`
