# 摄氏温度与华氏温度

你有没有去过美国或看过美国电影？那里的人不用摄氏度，而是用华氏度来表示温度。今天我们就当一次「温度翻译官」，用 Python 帮我们把摄氏温度转换成华氏温度！

## 概念讲解

在 Python 中，数字主要分为两种类型：

- **整数（int）**：像 0、1、-5、100 这样不带小数点的数。
- **浮点数（float）**：像 3.14、-0.5、98.6 这样带小数点的数。

当整数和浮点数一起做运算时，Python 会自动把结果变成浮点数。

### 算术运算符

Python 支持四种基本算术运算：

| 运算符 | 含义 | 例子 |
|--------|------|------|
| `+` | 加法 | `5 + 3` → `8` |
| `-` | 减法 | `10 - 4` → `6` |
| `*` | 乘法 | `6 * 7` → `42` |
| `/` | 除法 | `9 / 5` → `1.8` |

> **注意**：Python 3 中 `/` 永远返回浮点数，即使能整除。比如 `10 / 2` 的结果是 `5.0`。

### 运算优先级

和数学一样，Python 也遵守「先乘除、后加减」的规则。可以用小括号 `()` 改变计算顺序。

例如：`9 / 5 + 32` 会先算 `9 / 5`，再加 `32`。

## 代码示例

```python
# 示例 1：温度转换
celsius = 0
fahrenheit = celsius * 9 / 5 + 32
print(fahrenheit)  # 输出 32.0

# 示例 2：混合运算
print(10 + 3 * 2)  # 输出 16（先乘后加）
print((10 + 3) * 2)  # 输出 26（括号优先）
```

## 常见错误

1. **忘记运算优先级**：写 `celsius * 9 / 5 + 32` 时，Python 会从左到右依次计算，结果是正确的。但如果你写 `celsius * (9 / 5) + 32` 也是一样的——`9 / 5` 先算成 `1.8`，再乘 `celsius`。
2. **误用整数除法**：在其他语言中 `9 / 5` 可能得 `1`（截断小数），但在 Python 3 中 `/` 总是返回 `float`，所以 `9 / 5 = 1.8` 完全正确。

## 题目要求

编写一个函数 `py_learn(celsius)`，它接收一个数字（摄氏温度），返回对应的华氏温度。

**公式**：F = C × 9 / 5 + 32

- 输入：一个数字（整数或浮点数均可）
- 输出：浮点数形式的华氏温度

示例：

- `py_learn(0)` → `32.0`
- `py_learn(100)` → `212.0`
